## Project direction

Decision Kernel is the pivot of Governance OS into a broader verification and cost-governance layer for hybrid human-agent systems operating under incomplete information.

The core loop is:
1. extract signal from messy inputs
2. evaluate explicit policy
3. auto-clear routine cases
4. escalate true exceptions
5. score outcomes retrospectively
6. adapt trust, authority, and compensation over time

## Strategic positioning

Best-fit problem framing:
- 04 Verification without managers
- 07 Cost governance for agents

Plausible extension:
- 05 Coordination without companies

This project should be framed as an operational substrate, not as token theater and not as a finished capital instrument.

## Pivot from Governance OS

Governance OS proved a narrower version of the kernel in finance workflows:
- deterministic policy evaluation
- signal extraction with provenance
- exception routing
- replayability
- audit-grade evidence packs

Decision Kernel keeps those primitives and changes the governed object:
- from treasury and wealth workflows
- to hybrid human-agent work, milestone verification, agent spend, and authority routing

## Important formulation to preserve

The system can increase trust, authority, and compensation for human or agent decision-makers whose policies repeatedly produce superior realized outcomes.

## How to talk about it

Say:
- verification and cost-governance layer
- hybrid human-agent systems
- operational substrate
- policy evaluation, exception routing, and retrospective learning
- agent spend as a governed resource

Do not say:
- token model
- finished capital instrument
- generic governance platform
- replacement for firms
