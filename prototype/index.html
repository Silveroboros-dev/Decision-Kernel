Decision Kernel MVP

Verification layer for milestone-gated capital release in hybrid human-agent teams.

One narrow loop. A contributor submits a milestone claim, evidence is compressed into structured signal, policy and spend checks run, an exception is routed if needed, the decision is logged, and later outcomes rescore future trust.

Primitive

  • Problem#4 Verification without managers
  • Secondary proof#7 Cost governance for agents
  • Sponsor painCapital should move only when work is verified

Compressed primitive

  • If milestone evidence is incomplete, do not auto-clear.
  • If spend exceeds cap, escalate.
  • If release exceeds remaining budget, block.
  • If evidence and spend are clean, release capital automatically.

Operator View

Why this primitive matters

What moves

Before the decision, no capital moves. After the bounded approval, only $500 is released.

What is governed

  • Milestone evidence completeness
  • Blocking issue status
  • Agent and tool spend
  • Budget release amount
  • Future trust and authority

Who this is for

Protocol foundations, accelerators, grant programs, studios, and companies funding hybrid teams. The MVP is not a capital instrument. It is the verification layer those sponsors would need before capital can move programmatically.

Audit note

No recommendation is hidden. The escalation reason, bounded decision, and later outcome are all explicit and reviewable.
